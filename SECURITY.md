# Security Policy

## Supported Versions

The following table lists versions of **Smart Network Monitor AI** that currently receive security updates:

| Version | Supported          |
|---------|--------------------|
| 1.x     | ✅ Active support   |
| < 1.0   | ❌ No longer supported |

---

## Reporting a Vulnerability

We take security disclosures seriously. If you discover a vulnerability, please follow **responsible disclosure** principles:

### Please DO:

- Report privately **before** public disclosure
- Provide enough detail to reproduce the issue (steps, environment, PoC if safe to share)
- Allow reasonable time (up to **90 days**) to patch before public disclosure

### Please DO NOT:

- Open a public GitHub issue for security vulnerabilities
- Access, modify, or delete data that does not belong to you during testing
- Perform denial-of-service attacks

---

## How to Report

Send an email to **careed23@outlook.com** with the subject line:

```
[SECURITY] Smart Network Monitor AI — <brief description>
```

Include:

- A description of the vulnerability and its potential impact
- Steps to reproduce
- Any relevant log output, screenshots, or PoC code
- Your preferred contact method for follow-up

We will acknowledge receipt within **72 hours** and aim to provide a resolution timeline within **7 business days**.

---

## Disclosure Timeline

| Step | Target Timeframe |
|------|-----------------|
| Acknowledgement | 72 hours |
| Initial assessment | 7 business days |
| Patch or mitigation | Up to 90 days |
| Coordinated public disclosure | After patch is available |

---

## Scope

This policy covers the code in this repository. It does **not** cover third-party dependencies (report those to the respective upstream projects).

---

## Recognition

We appreciate responsible disclosure and are happy to acknowledge reporters in release notes (with your permission).
