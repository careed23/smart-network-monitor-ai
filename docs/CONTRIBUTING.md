# Contributing

> Note: The current repository is a frontend-only dashboard with simulated data. The documentation describes the target cloud architecture.

## How to Contribute
1. Fork the repository and create a feature branch.
2. Keep changes scoped and aligned with the reference architecture.
3. Open a pull request with a clear summary and testing notes.

## Documentation Updates
- Ensure architectural changes remain consistent across README and `/docs`.
- Keep cloud-architecture language concise and implementation-agnostic.

## Code Contributions
- Follow existing code style and keep UI behavior unchanged unless requested.
- Avoid adding dependencies without a security review.

## Pull Request Checklist
- [ ] Documentation updated (if applicable)
- [ ] Testing noted (or marked not run)
- [ ] Security/operational considerations documented
